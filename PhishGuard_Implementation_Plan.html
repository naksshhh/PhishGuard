Key insight: The cascade resolves ~55% of URLs at Tier 1 (benign, instant), ~30% after community cache, and ~10% at Tier 2. Only ~5% reach Gemini — keeping API costs manageable while maintaining SOTA accuracy on hard cases.

Contributions

4 Novel Claims (Publishable + Hackathon-Worthy)

All existing SOTA systems are research prototypes. None are real-time browser-deployable. Your four contributions address this gap directly.

01

Hierarchical Latency-Aware Cascade

Three-tier architecture matching model cost to signal uncertainty. 95% of URLs decided before any server call. No existing deployed extension does this.

02

True In-Browser ONNX Inference

INT8-quantized LightGBM running via WebAssembly inside the Chrome extension. No existing phishing extension does on-device ML inference at this tier.

03

Gemini Brand-Impersonation Reasoning

Multimodal LLM identifies the brand from a screenshot and verifies domain match — end-to-end deployed in a browser extension. No paper has done this.

04

Community Intelligence Layer

Crowd-sourced anonymous signals in Firebase. Zero-day emergent pattern detection via social consensus. No existing deployed system has this.

Skip for time: QML (Variational Quantum Classifier) from Gemini's roadmap and Neo4j threat graph are intellectually impressive but add ~3 days each. The cascade + community layer are already novel without them. Add QML only as a "future work" section in the capstone.
Execution Plan

15-Day Technical Roadmap

1
Foundation & Data Engineering Days 1–3
Critical Day 1 — Dataset construction (100k samples)
Merge PhishTank (~10k), OpenPhish free feed, Mendeley phishing dataset (50k phish + 30k legit w/ HTML), Kaggle UCI (11k), and Tranco top-1M for legitimate URLs. Build unified pipeline targeting >100k balanced corpus. Set up Weights & Biases for experiment tracking.
Claude Gemini
Critical Day 1 — Extract 40 engineered features
URL features (20): url_length, domain_length, n_subdomains, digit_ratio, special_char_count, entropy_domain, has_ip_as_domain, suspicious_keyword_count, tld_in_subdomain, brand_in_path, punycode_detected, redirect_count, https_present, url_depth, longest_path_token, avg_path_token_len, query_digit_count, dot_count, slash_count, ampersand_count.

HTML features (20): form_action_external, iframe_count, hidden_input_count, external_link_ratio, script_count, meta_refresh_present, login_form_present, password_field_count, title_domain_mismatch, favicon_external, anchor_to_javascript, copyright_year_present, social_media_links_count, page_size_bytes, whitespace_ratio, noscript_count, internal_link_count, image_count, ad_network_links, contact_page_link.
Claude
High Day 2 — GAN-based data augmentation
Use a Conditional GAN (CTGAN from the SDV library) to generate synthetic phishing samples from the tabular feature distribution. Target: 20k synthetic samples to balance rare attack categories. Validate with statistical similarity tests. This is a genuine SOTA technique from ChatGPT's insight.
ChatGPT Novel
High Day 2 — VAE for latent HTML feature extraction
Train a Variational Autoencoder on raw HTML content vectors. Extract the 32-dim latent representation as supplementary features for Stage 2. The KL-divergence objective forces the model to learn generalised phishing patterns, improving zero-day detection. Appended to the 40 engineered features.
Gemini
Medium Day 3 — Baseline model race (RF vs XGB vs LightGBM)
Run rapid experiment on combined 40-feature dataset. LightGBM wins on speed + export size — this becomes Stage 1. Target: ≥98.5% F1. Log all runs to W&B. Generate SHAP global feature importance plots for the capstone report.
Claude
Medium Day 3 — EGSO hyperparameter sweep
Run Enhanced Grid Search Optimization over LightGBM (n_estimators, max_depth, num_leaves, learning_rate, min_child_samples). Use Optuna for Bayesian optimization — faster than full grid. Target config from Gemini paper: achieves 99.44% via filters (64, 128, 256) and dropout 0.5.
Gemini
2
Core ML Pipeline — Stages 1 & 2 Days 4–7
Critical Day 3–4 — ONNX export + Chrome extension integration
Export trained LightGBM to ONNX via onnxmltools. Quantize to INT8 using onnxruntime.quantization (target size <300KB). Load in Chrome extension using onnxruntime-web. This is Stage 1 of the cascade — runs entirely on-device in <15ms.
Claude
Critical Day 4–5 — FastAPI backend + 3 async endpoints
Migrate Flask → FastAPI (async-native, auto OpenAPI docs). Three endpoints: POST /analyze/url (Stage 2, XGBoost + Safe Browsing API in parallel via asyncio.gather), POST /analyze/multimodal (Stage 3, Gemini), POST /feedback (community votes). SHAP explanation on every Stage 2 call.
Claude
High Day 5–6 — DistilBERT URL semantic encoder ("PhishBERT")
Fine-tune distilbert-base-uncased on URLs as character sequences (not word-piece). Tokenize by stripping scheme and treating URL as raw text. 3 epochs, lr=2e-5, AdamW on Colab T4 (~2hrs for 100k URLs). Critical value: catches homoglyph attacks like paypa1-secure-login.com that lexical features miss. Export to ONNX for serving.
Claude ChatGPT
High Day 6–7 — CodeBERT HTML structural branch
Implement and fine-tune CodeBERT (microsoft/codebert-base) on HTML source code. CodeBERT is pretrained on multiple programming languages, making it effective at detecting obfuscated JavaScript blocks, malicious form structures, and hidden iframes — all standard phishing evasion. Smart HTML excerpt: extract <head>, all <form>, <a> tags, <img> tags only (max 2000 chars).
Gemini ChatGPT
Novel Day 7 — Attention-weighted meta-classifier fusion
ChatGPT's contribution over simple weighted voting: train a small attention layer on top of the 4 branch probability scores. The meta-classifier learns which branch to trust more per scenario — if HTML is heavily obfuscated, it up-weights the visual branch. Implemented as a 2-layer MLP with a softmax attention head. More principled than Gemini's hard-coded weights.
ChatGPT Gemini
3
Multi-Modal & Visual Intelligence Days 8–10
Critical Day 8 — Gemini 1.5 Flash multimodal pipeline
This is your key hackathon differentiator. Use Gemini 1.5 Flash (not Pro — 3× faster, cheaper). System prompt returns structured JSON: identified_brand, credential_fields_present, domain_matches_brand, suspicion_score (0–1), verdict, plain-English explanation (shown directly to user), and evidence list. Set response_mime_type: "application/json" for reliable parsing.
Claude Novel
Critical Day 8–9 — Screenshot capture in Chrome extension
Use chrome.tabs.captureVisibleTab() at 60% quality JPEG. Crop to top 800px (phishing elements always appear above the fold). Convert to base64 for API transport. Also implement extractSmartHTMLExcerpt() — head innerHTML (500 chars) + all form elements + first 20 <a> tags + img tags, capped at 2000 chars total.
Claude
High Day 9 — EfficientNet-B7 visual branch
Train EfficientNet-B7 (transfer learning from ImageNet) on page screenshot dataset from Mendeley phishing dataset. Task: binary classification on cropped 800px screenshots. Use Grad-CAM to generate visual explanations of logo-cloning detections — these become demo visuals for the hackathon presentation. Target: >95% F1 on screenshot-only task.
Gemini
High Day 9–10 — SHAP + plain-English XAI pipeline
Integrate TreeExplainer (SHAP) for Stage 2. Map top-3 feature names to human-readable sentences: e.g., "has_ip_as_domain → Domain is an IP address instead of a name". This reduces alert fatigue vs. raw security warnings and builds user cyber-literacy. Integrate with Stage 3's Gemini explanation for a unified "why" layer.
Claude Gemini
Novel Day 10 — Adversarial test set construction
Build 200 adversarial URLs with: homoglyph substitution (paypa1.com, arnazon.in), URL padding attacks (/legitimate-path/real-bank-login), punycode encoding, and brand keywords in subpaths. Run cascade against this set — the visual branch should still catch them via logo detection regardless of URL manipulation. This table becomes a key result in your paper.
Novel
4
Community Layer & Deployment Days 11–13
Novel Day 11 — Firebase community intelligence layer
Set up Firebase Realtime Database (free Spark plan). Structure: domains/{base64(domain)} → { phishing_votes: int, legitimate_votes: int }. Trust threshold: ≥10 votes AND >80% consensus. User feedback via opt-in button. This zero-day emergent detection layer has no equivalent in any existing deployed system — it's your fourth novel contribution and a strong demo talking point.
Claude Novel
Critical Day 11–12 — Cascade orchestration wiring
Wire all 4 tiers together in the Chrome extension background service worker. Logic: (1) check community cache → instant if consensus. (2) Stage 1 ONNX → resolve if score <0.35 or >0.75. (3) Stage 2 FastAPI → resolve if score <0.35 or >0.65. (4) Stage 3 Gemini → final verdict. All async, with timeout fallbacks at each stage. Show which stage resolved — demo talking point: "95% decided in <15ms".
Claude
High Day 12 — Docker + Google Cloud Run deployment
Containerize FastAPI backend. Download models at build time (not runtime). Deploy to Cloud Run (free tier: 2M requests/month). Set memory to 2Gi, timeout 30s. Cloud Run = Google tech = extra hackathon points. Use LLVM model compilation for 26% inference boost. Total backend latency target: <500ms end-to-end.
Claude
High Day 12–13 — Email phishing detector upgrade
Replace Naive Bayes with Gemini API for email analysis (handles multi-part threads, intent reasoning on urgency/pressure language). JSON response: urgency_language, suspicious_links, impersonated_brand, credential_request, verdict, risk_score, plain-English explanation. The sentiment analysis → risk scoring (0–5) from Gemini's MemoPhishAgent paper applies here.
Gemini ChatGPT
Medium Day 13 — Popup UI redesign
Circular risk gauge (CSS-only). Prominent plain-English explanation. Tier badge showing which stage resolved ("⚡ Edge — 12ms"). Report button for false positives/negatives (feeds Firebase). Green/amber/red toolbar shield icon with color coding. Key principle: show non-technical users a clear verdict, not raw numbers.
Claude

Technology

Full Technology Stack

Browser Extension

Chrome MV3 + ONNX Runtime Web

On-device inference via WebAssembly. No latency. INT8-quantized LightGBM (<300KB). Firebase SDK for community layer.

Stage 2 Backend

FastAPI + Python 3.11

Async-native, auto-generates OpenAPI docs (professional for submission). Replaces Flask. asyncio.gather for parallel branch execution.

Stage 2 ML — Tabular

XGBoost + LightGBM + SHAP

40-feature tabular classifier. SHAP TreeExplainer for top-3 feature attribution → plain-English XAI for user.

Stage 2 ML — Semantic

DistilBERT (fine-tuned) + CodeBERT

"PhishBERT" for URL character sequences. CodeBERT for HTML source analysis. Both exported to ONNX for serving.

Stage 2 ML — Visual (Edge)

EfficientNet-B7 + Grad-CAM

Screenshot classification. Grad-CAM generates logo-cloning visualisations for demo + explanations.

Stage 3 — Reasoning

Gemini 1.5 Flash API

Multimodal: screenshot + HTML → brand identification + domain verification. JSON response with plain-English user explanation.

Feature Engineering

VAE + CTGAN (SDV library)

VAE for latent HTML features (zero-day generalisation). CTGAN for synthetic phishing sample augmentation.

Community Layer

Firebase Realtime Database

Free Spark plan. Real-time, globally distributed. Anonymous opt-in voting. 80% consensus threshold.

Deployment

Google Cloud Run

Serverless, scales to zero. 2M requests/month free. Google tech = hackathon alignment points.

Optimization

LLVM Model Compilation

26% production inference boost on Cloud Run. INT8 quantization for ONNX models. asyncio parallel branch execution.

Blacklist Layer

Google Safe Browsing API

Free, high coverage. Runs in parallel with Stage 2 ML classifier. Combined score: 0.7×ML + 0.3×SafeBrowsing.
